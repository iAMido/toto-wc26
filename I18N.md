# i18n & RTL — toto-wc26

Two languages: **Hebrew (RTL, default)** and **English (LTR)**. Switching is instant — no page reload.

## Stack

- `i18next` + `react-i18next` for translation lookup and React bindings.
- A custom `RtlProvider` component that updates `<html dir>` and `<html lang>` whenever the active language changes.
- Tailwind logical properties (`ms-`, `me-`, `ps-`, `pe-`, `text-start`, `text-end`, `start-0`, `end-0`) for layouts that mirror automatically.

## Default & detection

- First-launch default: **`he`** (Hebrew, `dir=rtl`).
- On subsequent launches, restore the user's saved choice from `users.locale` (or `localStorage` for unauthenticated visitors).
- Users can switch via a header dropdown; the change is persisted to `users.locale` and `localStorage`.

## File layout

```
src/i18n/
├── i18n.ts            # i18next init, resource registry, defaultNS = 'common'
├── RtlProvider.tsx    # <html dir lang> sync + listens to i18next.languageChanged
├── en.json            # English keys
└── he.json            # Hebrew keys (identical key set)
```

## Key naming conventions

- Use **nested namespaces by feature**, not flat: `nav.signOut`, `match.predictionLocked`, `joker.remaining`.
- Keys are English even in `he.json` — the value is translated, the key is not.
- Use ICU plural forms for counts: `joker.remaining_one`, `joker.remaining_other`.
- Interpolation: `{{count}}`, `{{teamName}}` — always wrap dynamic values in curlies.

Example:

```json
// en.json
{
  "joker": {
    "remaining_one": "{{count}} Joker remaining",
    "remaining_other": "{{count}} Jokers remaining",
    "capReached": "All 3 Jokers used"
  }
}
```

```json
// he.json
{
  "joker": {
    "remaining_one": "נותר ג'וקר אחד",
    "remaining_other": "נותרו {{count}} ג'וקרים",
    "capReached": "כל 3 הג'וקרים נוצלו"
  }
}
```

## RTL rules

1. **Always use logical Tailwind utilities** instead of `left`/`right`:
   - `ms-2` not `ml-2`
   - `text-start` not `text-left`
   - `border-s` not `border-l`
2. Icons that imply direction (back arrow, "next" chevron) must flip with the layout. Wrap in a `<span class="rtl:scale-x-[-1]">`.
3. Numbers stay in Western Arabic (`0–9`) in both languages — do not force Hebrew numerals.
4. Football team names and player names are stored bilingually (`name_en` / `name_he`); render the one matching the active locale.
5. Dates: `Intl.DateTimeFormat(locale, ...)` — produces correct HE month names automatically.

## Bidirectional content

Predicted scores like "2:1" are direction-neutral. To avoid the browser reordering them in RTL contexts, wrap in `<bdi>`:

```tsx
<bdi>{home}:{away}</bdi>
```

## Tailwind config snippet

```ts
// tailwind.config.ts
export default {
  // ...
  theme: {
    extend: {
      // (logical properties are built-in in modern Tailwind)
    }
  }
}
```

Tailwind v3.3+ supports the `rtl:` variant out of the box — use sparingly for cases where logical properties aren't enough (e.g. icon flipping).

- [ ] Switch to English → `<html dir="ltr" lang="en">` immediately, no reload.
